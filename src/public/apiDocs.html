Finance Manager

API Reference

A RESTful Node.js + Express API for personal finance management. Supports role-based access control, JWT cookie authentication, and full CRUD for financial records.

Runtime
Node.js ESM
Auth
JWT Cookie
Database
MongoDB
🔐
Authentication & Roles
JWT cookie-based sessions with role-based authorization
Authentication Flow
The API uses JWT tokens stored in an HTTP-only cookie named token. Tokens expire in 2 minutes. To authenticate, call POST /login. The token is automatically sent with subsequent requests.
Localhost Base URL: http://localhost:{PORT}
Vercel Base URL: https://finance-manager-zeta-weld.vercel.app/
Roles
  • admin — Full access. Create/delete users, manage records, change roles.
  • analyst — Read-only access to records and dashboard.
  • viewer — Can only delete their own account. Registered via public endpoint.
  • public — No auth required (login, register viewer).
Standard Error Format
JSON
// Validation errors (400)
[{ "msg": "Amount should not be empty.",
   "path": "amount" }]

// Auth / not found errors
"Invalid credentials"
"Unauthorized"
Content Type
All request bodies must be sent as application/json. All responses return application/json. Dates must be formatted as YYYY-MM-DD.

Authentication
Login, logout, and session info
POST /login Authenticate user Public ▶

Authenticates a user with email and password. Sets a JWT token in an HTTP-only cookie on success. Returns 403 if the user is already logged in with a valid token.

Request Body
FieldTypeRequiredDescription
email_idstringRequiredValid email address
passwordstringRequiredMin 8 characters
Example Request
JSON
{
  "email_id": "admin@company.com",
  "password": "securepass123"
}
Responses
200"Login successful" — cookie set
400Validation errors array
401"Invalid credentials"
403"User already logged in"
500Internal server error
DELETE /logout End session JWT Cookie ▶

Ends the user's session by verifying the JWT token, recording logout time, and clearing the token cookie. Returns 401 if no valid token is present.

Responses
200"Logged out successfully" — cookie cleared
401"Unauthorized" or "Token expired or invalid"
500Internal server error
GET /me Get current user profile JWT Cookie ▶

Returns the profile of the currently authenticated user based on the JWT token in the cookie.

Responses
200 User object: { email_id, name, role, login_time, logout_time, last_active }
401"Unauthorized" or "Token expired or invalid"
500Internal server error

POST /user Create user (admin) Admin ▶

Creates a new user with any role. Only accessible by admins.

Request Body
FieldTypeRequiredDescription
email_idstringRequiredValid, unique email address
namestringRequiredNon-empty display name
passwordstringRequiredMin 8 characters
rolestringOptionalviewer | analyst | admin (defaults to viewer)

JSON

}
Responses

400Validation errors
403Forbidden — admin only

Allows viewer or analyst users to permanently delete their own account. Admins are excluded from this endpoint (they must use another admin to delete them). The user is identified from the JWT cookie.

Records
Financial transaction management — income and expense records
GET /records List records (filtered, paginated) AdminAnalyst ▶

Returns a paginated, filterable list of financial records. Supports filtering by transaction type, category, and date range.

Query Parameters
ParameterTypeRequiredDescription
typestringOptionalincome or expense
categorystringOptionalFilter by category name
startDatestringOptionalFilter from date — YYYY-MM-DD
endDatestringOptionalFilter to date — YYYY-MM-DD
pageintegerOptionalPage number
limitintegerOptionalRecords per page
Example URL
URL
GET /records?type=expense&category=food&startDate=2024-01-01&page=1&limit=20
Responses
200Array of record objects with pagination info
400Invalid query parameter format
403Forbidden — requires analyst or admin

Dashboard
Aggregated financial summaries — read-only, all authenticated users
GET /total/income Total income across all records Any authenticated ▶

Returns the sum of all records where type = "income".

Example Response
JSON
{ "totalIncome": 125000 }
Responses
200{ totalIncome: number }
401Unauthorized
500Internal server error
GET /total/expense Total expense across all records Any authenticated ▶

Returns the sum of all records where type = "expense".

Example Response
JSON
{ "totalExpense": 47500 }
Responses
200{ totalExpense: number }
401Unauthorized
500Internal server error
GET /total Net balance (income minus expenses) Any authenticated ▶

Returns the net balance: the sum of all income records minus the sum of all expense records. Computed in a single MongoDB aggregation pipeline.

Example Response
JSON
{ "netBalance": 77500 }
Responses
200{ netBalance: number }
401Unauthorized
500Internal server error
GET /categories Income & expense totals by category Any authenticated ▶

Returns an array of category-level aggregations. Each entry shows the total income and/or expense for that category. Fields are omitted when zero.

Example Response

    "totalIncome": 120000

  {
    "category": "food",
    "totalExpense": 18500
  },
  {
    "category": "freelance",
    "totalIncome": 5000,
    "totalExpense": 2200
  }
]
Responses
200Array of { category, totalIncome?, totalExpense? }
401Unauthorized
500Internal server error
